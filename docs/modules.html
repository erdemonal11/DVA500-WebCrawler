webcrawler.client.check_visited(url, host, port, process_id)[source]

Check if a URL has already been visited by querying the server.

Connects to the server via socket and sends the URL. The server responds with ‘Y’ if the URL was visited, ‘N’ otherwise.

Parameters:

  • url (str) – The URL to check.

  • host (str) – Server hostname or IP address.

  • port (int) – Server port number.

  • process_id (str) – Process identifier for logging purposes.

Returns:

True if the URL was already visited, False otherwise.

Return type:

bool

Raises:

SystemExit – If connection to the server fails.

webcrawler.client.main()[source]

Main entry point for the web crawler client.

Sets up logging, reads configuration, and visits the specified URL. Logs all operations and exits after completion.

Command-line arguments:

sys.argv[1]: Process ID for identification. sys.argv[2]: URL to visit.

Environment variables:

LOG_LEVEL: Logging level (DEBUG, INFO, WARNING, ERROR). Defaults to INFO.

webcrawler.client.request_with_retry(url, retries=5, delay=5, process_id=None)[source]

Request a URL with retry mechanism.

Attempts to fetch a URL with automatic retry on connection errors. Logs warnings for each failed attempt and raises an exception if all retries fail.

Parameters:

  • url (str) – The URL to request.

  • retries (int) – Number of retry attempts. Defaults to 5.

  • delay (int) – Delay between retries in seconds. Defaults to 5.

  • process_id (str, optional) – Process identifier for logging purposes.

Returns:

Response object if the request succeeds.

Return type:

requests.Response

Raises:

Exception – If all retry attempts fail to connect to the URL.

webcrawler.client.visit(url, host, port, process_id)[source]

Visit a URL if it hasn’t been visited before.

Checks with the server if the URL was already visited. If not, fetches the URL content and parses it with BeautifulSoup.

Parameters:

  • url (str) – The URL to visit.

  • host (str) – Server hostname or IP address.

  • port (int) – Server port number.

  • process_id (str) – Process identifier for logging purposes.

Returns:

This function returns nothing.

Return type:

None

webcrawler.server.main()[source]

Main entry point for the web crawler server.

Sets up a socket server that tracks visited URLs. Clients send URLs and receive ‘Y’ if visited, ‘N’ otherwise. Logs all operations and statistics on shutdown.

Configuration:

Reads server port from crawler.cfg file.

Environment variables:

LOG_LEVEL: Logging level (DEBUG, INFO, WARNING, ERROR). Defaults to INFO.

Signals:

Handles KeyboardInterrupt (CTRL+C) gracefully, logging statistics before exit.
